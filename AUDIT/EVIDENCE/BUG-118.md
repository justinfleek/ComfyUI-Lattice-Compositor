# BUG-118: VideoLayer.setAudioLevel() NaN Bypass

## Summary
The `setAudioLevel(level)` method uses `Math.max(0, Math.min(1, level / 100))` to clamp the volume. However, if `level` is NaN, the division and clamping operations all return NaN, which is then assigned to `videoElement.volume`.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/VideoLayer.ts
- Lines: 532-537
- Function: setAudioLevel

## The Bug
```typescript
// Lines 532-537
setAudioLevel(level: number): void {
  this.videoData.audioLevel = level;  // No validation
  if (this.videoElement) {
    this.videoElement.volume = Math.max(0, Math.min(1, level / 100));
    //                         ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    // NaN / 100 = NaN
    // Math.min(1, NaN) = NaN
    // Math.max(0, NaN) = NaN
    // volume = NaN (invalid)
  }
}
```

## Proof of Failure
```typescript
const videoLayer = new VideoLayer(layerData, resources);
await videoLayer.loadVideo(assetId);

// NaN behavior:
console.log(NaN / 100);           // NaN
console.log(Math.min(1, NaN));    // NaN
console.log(Math.max(0, NaN));    // NaN

// Attack: level = NaN
videoLayer.setAudioLevel(NaN);
// videoElement.volume = NaN
// Browser behavior: may throw, may set to 0, may ignore
// Audio playback becomes unpredictable

// Also stores NaN in videoData:
// this.videoData.audioLevel = NaN
// Persisted to project file if saved
```

## Impact
- Audio volume becomes undefined
- NaN stored in videoData may be serialized to project
- Browser may throw or have undefined volume behavior
- Low severity: Usually set from UI slider with valid values

## Suggested Fix
```typescript
setAudioLevel(level: number): void {
  // Guard against NaN
  if (!Number.isFinite(level)) {
    level = 100;  // Default to full volume
  }

  this.videoData.audioLevel = level;
  if (this.videoElement) {
    this.videoElement.volume = Math.max(0, Math.min(1, level / 100));
  }
}
```
