# BUG-142: updateAnimation() NaN Speed Propagation

## Summary
updateAnimation() reads animation.speed without validation. NaN or Infinity speed values corrupt the animation mixer update.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/ModelLayer.ts
- Lines: 525-530
- Function: updateAnimation()

## Crash Path Trace
```
Function: updateAnimation(deltaTime: number)

Line 528-529:
  const speed = this.modelData.animation?.speed ?? 1;
  this.mixer.update(deltaTime * speed);

  - speed=NaN → deltaTime * NaN = NaN → mixer.update(NaN) → animation corrupted
  - speed=Infinity → deltaTime * Infinity = Infinity → animation jumps to end
  - speed=0 → deltaTime * 0 = 0 → animation frozen (may be intentional)

Result: SILENT CORRUPTION - animation playback corrupted
```

## Code Evidence
```typescript
// Line 525-530
private updateAnimation(deltaTime: number): void {
  if (!this.mixer) return;

  const speed = this.modelData.animation?.speed ?? 1;  // NaN passes through
  this.mixer.update(deltaTime * speed);  // Corrupted update
}
```

## Impact
- Animation playback becomes erratic
- User sets speed to NaN via updateLayerData → animation broken
- Infinity speed makes animation unusable

## Suggested Fix
```typescript
private updateAnimation(deltaTime: number): void {
  if (!this.mixer) return;

  const speed = this.modelData.animation?.speed ?? 1;
  if (!Number.isFinite(speed)) {
    console.warn(`updateAnimation: invalid speed ${speed}, using 1`);
    this.mixer.update(deltaTime);
    return;
  }
  this.mixer.update(deltaTime * speed);
}
```
