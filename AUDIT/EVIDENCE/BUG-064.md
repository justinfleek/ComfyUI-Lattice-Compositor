# BUG-064: Infinite Loop When maxParticles is Infinity (ParticleCollisionSystem)

## Summary
Same vulnerability pattern as BUG-059, BUG-061, BUG-062. The ParticleCollisionSystem has two loops bounded by maxParticles. If maxParticles is Infinity, both loops run forever.

## Severity: LOW

## Location
- File: /ui/src/engine/particles/ParticleCollisionSystem.ts
- Lines: 101, 255
- Functions: applyBoundaryCollisions, applyParticleCollisions

## The Bug
```typescript
// Line 101 - applyBoundaryCollisions
for (let i = 0; i < this.maxParticles; i++) {
  // If maxParticles = Infinity, never terminates
}

// Line 255 - applyParticleCollisions
for (let i = 0; i < this.maxParticles; i++) {
  // Same issue
}
```

## Proof of Failure
```typescript
const collision = new ParticleCollisionSystem(
  Infinity,  // maxParticles
  {
    enabled: true,
    boundsBehavior: 'bounce',
    bounds: { min: { x: 0, y: 0, z: 0 }, max: { x: 100, y: 100, z: 100 } }
  }
);

collision.update(particleBuffer); // Never returns - infinite loop
```

## Related
- BUG-059: Same pattern in SpatialHashGrid.rebuild()
- BUG-061: Same pattern in ParticleConnectionSystem.update()
- BUG-062: Same pattern in ParticleTrailSystem.update()

## Impact
- Browser freeze when maxParticles = Infinity
- Low severity: Requires specific bad input

## Suggested Fix
```typescript
constructor(maxParticles: number, config: Partial<CollisionConfig> = {}) {
  if (!Number.isFinite(maxParticles) || maxParticles < 0) {
    console.warn('ParticleCollisionSystem: Invalid maxParticles, using 10000');
    this.maxParticles = 10000;
  } else {
    this.maxParticles = maxParticles;
  }
  // ... rest of constructor
}
```
