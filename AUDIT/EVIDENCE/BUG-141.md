# BUG-141: setAnimationTime() No Input Validation

## Summary
setAnimationTime() passes time value directly to Three.js AnimationAction without validation. NaN, negative, and Infinity values corrupt animation state.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/ModelLayer.ts
- Lines: 516-520
- Function: setAnimationTime()

## Crash Path Trace
```
Function: setAnimationTime(time: number)

Line 518-519:
  this.currentAction.time = time;  // NaN/negative/Infinity passed directly
  this.mixer.update(0);

  - time=NaN → currentAction.time = NaN → animation at undefined position
  - time=-100 → currentAction.time = -100 → negative time (undefined behavior)
  - time=Infinity → currentAction.time = Infinity → animation at end/broken

Result: SILENT CORRUPTION - animation state corrupted
```

## Code Evidence
```typescript
// Line 516-520
setAnimationTime(time: number): void {
  if (!this.mixer || !this.currentAction) return;
  this.currentAction.time = time;  // No validation
  this.mixer.update(0);
}
```

## Impact
- Animation scrubbing produces garbage output
- User sets keyframe to NaN time → animation broken
- Negative time produces undefined Three.js behavior

## Suggested Fix
```typescript
setAnimationTime(time: number): void {
  if (!this.mixer || !this.currentAction) return;

  if (!Number.isFinite(time) || time < 0) {
    console.warn(`setAnimationTime: invalid time ${time}`);
    return;
  }

  // Clamp to animation duration
  const duration = this.currentAction.getClip().duration;
  this.currentAction.time = Math.min(time, duration);
  this.mixer.update(0);
}
```
