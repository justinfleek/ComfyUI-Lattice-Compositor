# BUG-116: VideoLayer.setFPS() No Validation - Division by Zero

## Summary
The `setFPS(fps)` method accepts the fps parameter without validation and stores it directly. This value is used as a divisor in `calculateVideoTime()`, causing division by zero when fps=0.

## Severity: MEDIUM

## Location
- File: /ui/src/engine/layers/VideoLayer.ts
- Lines: 556-558 (setFPS), 492 (division)
- Functions: setFPS, calculateVideoTime

## The Bug
```typescript
// Lines 556-558
setFPS(fps: number): void {
  this.compositionFPS = fps;  // No validation!
}

// Line 492 in calculateVideoTime():
let videoTime = (layerFrame / compFps) * effectiveSpeed;
//              ^^^^^^^^^^^^^^^^^^^^ Division by compFps (this.compositionFPS)
// If fps=0: layerFrame / 0 = Infinity
```

## Proof of Failure
```typescript
const videoLayer = new VideoLayer(layerData, resources);

// Attack 1: fps = 0
videoLayer.setFPS(0);
videoLayer.seekToFrame(100);
// In calculateVideoTime:
// videoTime = (100 / 0) * 1 = Infinity
// seekToFrame attempts to seek to Infinity seconds
// Video element: currentTime = Infinity (undefined behavior)

// Attack 2: fps = NaN
videoLayer.setFPS(NaN);
videoLayer.seekToFrame(100);
// videoTime = (100 / NaN) * 1 = NaN
// Video element: currentTime = NaN (undefined behavior)

// Attack 3: fps = -30
videoLayer.setFPS(-30);
videoLayer.seekToFrame(100);
// videoTime = (100 / -30) * 1 = -3.33...
// Negative time - video seek may fail or behave unexpectedly
```

## Impact
- Division by zero produces Infinity video time
- NaN fps produces NaN video time
- Video playback becomes completely broken
- Medium severity: Usually called from engine with composition fps, but external calls possible

## Related
- BUG-090: BaseLayer.setCompositionFps same pattern
- BUG-102: LatticeEngine.setCompositionFPS same pattern
- SYSTEMIC-003: Division by zero / fps=0 pattern
- 14A: Numeric inputs - 0, NaN, negative testing

## Suggested Fix
```typescript
setFPS(fps: number): void {
  if (!Number.isFinite(fps) || fps <= 0) {
    console.warn('VideoLayer.setFPS: Invalid fps, using 30');
    fps = 30;
  }
  this.compositionFPS = fps;
}
```
