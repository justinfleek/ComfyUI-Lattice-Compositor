# BUG-161: LightLayer.updatePathFollowing() NaN Bypass in Clamp

## Summary
updatePathFollowing() uses Math.max/min to clamp progress with audio modifier, but NaN bypasses these guards. NaN progress passed to path provider.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/LightLayer.ts
- Lines: 591-600
- Function: updatePathFollowing()

## Crash Path Trace
```
Function: updatePathFollowing(frame: number)

Line 591: let progress = this.evaluator.evaluate(path.progress, frame);
Line 595-598:
  const audioMod = this.currentAudioModifiers;
  if (audioMod.pathPosition !== undefined && audioMod.pathPosition !== 0) {
    progress = Math.max(0, Math.min(1, progress + audioMod.pathPosition));
  }

Test case: audioMod.pathPosition = NaN
  - audioMod.pathPosition !== 0 → NaN !== 0 → true (check passes)
  - progress + NaN = NaN
  - Math.min(1, NaN) = NaN
  - Math.max(0, NaN) = NaN
  - progress = NaN

Line 600: const result = this.pathProvider(path.splineLayerId, progress, frame);
  - progress=NaN passed to spline lookup
  - Spline returns undefined/NaN position

Line 605: this.group.position.set(result.point.x, -result.point.y, result.point.z);
  - If result.point contains NaN → position becomes NaN

Result: SILENT CORRUPTION (SYSTEMIC-005) - light position becomes NaN
```

## Code Evidence
```typescript
// Lines 591-605
private updatePathFollowing(frame: number): void {
  // ...
  let progress = this.evaluator.evaluate(path.progress, frame);

  const audioMod = this.currentAudioModifiers;
  if (audioMod.pathPosition !== undefined && audioMod.pathPosition !== 0) {
    progress = Math.max(0, Math.min(1, progress + audioMod.pathPosition));  // NaN bypasses
  }

  const result = this.pathProvider(path.splineLayerId, progress, frame);  // NaN passed
  // ...
  this.group.position.set(result.point.x, -result.point.y, result.point.z);
}
```

## Impact
- Audio analysis can produce NaN values
- NaN !== 0 is true, so check doesn't filter NaN
- Math.max/min don't catch NaN (SYSTEMIC-005)
- Light position becomes NaN, disappears from scene
- Same pattern as BUG-156 (DepthflowLayer audio NaN bypass)

## Suggested Fix
```typescript
if (audioMod.pathPosition !== undefined && Number.isFinite(audioMod.pathPosition) && audioMod.pathPosition !== 0) {
  progress = Math.max(0, Math.min(1, progress + audioMod.pathPosition));
}
```
