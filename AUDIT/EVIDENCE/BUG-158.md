# BUG-158: LightLayer.setIntensity() No Validation

## Summary
setIntensity() stores intensity value and sets light.intensity without validation. NaN/Infinity values corrupt lighting calculations.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/LightLayer.ts
- Lines: 650-654
- Function: setIntensity()

## Crash Path Trace
```
Function: setIntensity(intensity: number)

Line 651: this.lightData.intensity = intensity;  // NaN stored
Line 652: this.lightData.usePhysicalIntensity = false;
Line 653: this.light.intensity = intensity / 100;  // NaN/100 = NaN

Test cases:
  - intensity=NaN → light.intensity=NaN → lighting calculations produce NaN
  - intensity=Infinity → light.intensity=Infinity → over-bright rendering
  - intensity=-100 → light.intensity=-1 → negative intensity (undefined behavior)

Result: SILENT CORRUPTION - light intensity invalid
```

## Code Evidence
```typescript
// Lines 650-654
setIntensity(intensity: number): void {
  this.lightData.intensity = intensity;  // No validation
  this.lightData.usePhysicalIntensity = false;
  this.light.intensity = intensity / 100;  // NaN propagates
}
```

## Impact
- NaN intensity causes lighting calculations to fail
- Infinity causes blown-out scene
- Negative intensity has undefined Three.js behavior
- Scene lighting becomes unpredictable

## Suggested Fix
```typescript
setIntensity(intensity: number): void {
  if (!Number.isFinite(intensity) || intensity < 0) {
    console.warn(`setIntensity: invalid intensity ${intensity}`);
    return;
  }
  this.lightData.intensity = intensity;
  // ...
}
```
