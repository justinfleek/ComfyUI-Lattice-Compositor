# BUG-097: GPUParticleSystem.simulateToFrame() Division by Zero

## Summary
`simulateToFrame(targetFrame, fps)` calculates `deltaTime = 1 / fps` without validating fps. When fps=0, deltaTime becomes Infinity, causing particles to move infinite distances per simulation step.

## Severity: MEDIUM

## Location
- File: /ui/src/engine/particles/GPUParticleSystem.ts
- Lines: 1500-1502
- Function: simulateToFrame

## The Bug
```typescript
// Lines 1500-1502
simulateToFrame(targetFrame: number, fps: number = 16): number {
  if (!this.frameCacheSystem) return 0;
  const deltaTime = 1 / fps;  // DIVISION BY fps!
  // ...
  this.step(deltaTime);  // deltaTime = Infinity when fps = 0
```

## Proof of Failure
```typescript
const system = new GPUParticleSystem(config);
system.initialize(renderer);

// fps = 0 passed from ParticleLayer (via BUG-085)
system.simulateToFrame(100, 0);
// -> deltaTime = 1 / 0 = Infinity
// -> step(Infinity) called
// -> Particles move infinite distance: px += vx * Infinity = Infinity
// -> All particles teleport to Infinity, simulation corrupted

// fps = NaN
system.simulateToFrame(100, NaN);
// -> deltaTime = 1 / NaN = NaN
// -> step(NaN) called
// -> px += vx * NaN = NaN
// -> All particle positions become NaN
```

## Impact
- fps=0: All particles teleport to infinity, visual corruption
- fps=NaN: All particle positions become NaN, invisible/broken
- Simulation state corrupted beyond recovery without reset

## Related
- BUG-085: ParticleLayer.setFPS() passes invalid fps upstream
- BUG-090: BaseLayer.setCompositionFps() same pattern
- SYSTEMIC-003: Division by Zero / fps=0 pattern

## Suggested Fix
```typescript
simulateToFrame(targetFrame: number, fps: number = 16): number {
  if (!this.frameCacheSystem) return 0;

  // Validate fps to prevent division by zero
  if (!Number.isFinite(fps) || fps <= 0) {
    console.warn('GPUParticleSystem.simulateToFrame: Invalid fps, using 30');
    fps = 30;
  }

  const deltaTime = 1 / fps;
  // ... rest of implementation
}
```
