# BUG-060: TypeError When SubEmitter overrides is Undefined

## Summary
The processDeathEvents function accesses `subEmitter.overrides.initialSpeed` without checking if `overrides` is defined. If a SubEmitterConfig has undefined overrides, this throws TypeError.

## Severity: LOW

## Location
- File: /ui/src/engine/particles/ParticleSubEmitter.ts
- Lines: 158-159
- Function: processDeathEvents

## The Bug
```typescript
// Lines 158-159
const overrides = subEmitter.overrides;  // No null check
const speed = overrides.initialSpeed ?? 100;  // TypeError if overrides undefined

// Also affects lines 160, 177, 180, 181, 185, 188 - all access overrides.X
```

## Proof of Failure
```typescript
const subEmitter = new ParticleSubEmitter(1000, Math.random);

subEmitter.addSubEmitter({
  id: 'explosion',
  trigger: 'death',
  parentEmitterId: '*',
  triggerProbability: 1,
  emitCount: 5,
  emitCountVariance: 0,
  inheritVelocity: 0.5,
  inheritSize: 0,
  inheritColor: 0,
  inheritRotation: 0,
  // overrides: undefined - NOT PROVIDED
});

subEmitter.queueDeathEvent({ index: 0 });

// This throws TypeError: Cannot read properties of undefined (reading 'initialSpeed')
subEmitter.processDeathEvents(particleBuffer, [1, 2, 3, 4, 5]);
```

## Impact
- Particle death effects crash if SubEmitterConfig missing overrides
- Uncaught exception could crash render loop
- Low severity: Requires malformed config, TypeScript should catch if types are strict

## Suggested Fix
```typescript
// Add guard at line 158
const overrides = subEmitter.overrides ?? {};
const speed = overrides.initialSpeed ?? 100;
// ... rest of accesses are now safe due to ?? defaults
```

Or add a validation in addSubEmitter:
```typescript
addSubEmitter(config: SubEmitterConfig): void {
  // Ensure overrides exists
  if (!config.overrides) {
    config.overrides = {};
  }
  this.subEmitters.set(config.id, config);
}
```
