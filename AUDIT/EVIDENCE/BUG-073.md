# BUG-073: Division by Zero When fontSize is 0

## Summary
In TextLayer.createTextMesh(), the outline width is calculated as `strokeWidth / fontSize`. If fontSize is explicitly set to 0, this produces Infinity, corrupting the text rendering.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/TextLayer.ts
- Line: 359
- Function: createTextMesh

## The Bug
```typescript
// Line 232 - fontSize defaults use nullish coalescing (doesn't catch 0)
fontSize: data?.fontSize ?? 72,

// Line 358-360 - Division by fontSize
if (this.textData.stroke && this.textData.strokeWidth > 0) {
  text.outlineWidth = this.textData.strokeWidth / this.textData.fontSize;  // fontSize=0 → Infinity!
  text.outlineColor = this.textData.stroke;
}
```

Nullish coalescing (`??`) doesn't catch 0:
- `undefined ?? 72` = 72 ✓
- `null ?? 72` = 72 ✓
- `0 ?? 72` = 0 ✗ (fontSize stays 0)

## Proof of Failure
```typescript
const textLayer = new TextLayer({
  data: {
    text: 'Hello',
    fontSize: 0,  // Explicit 0
    stroke: '#000000',
    strokeWidth: 2
  }
}, resources);

// Line 232: fontSize = 0 ?? 72 = 0
// Line 359: outlineWidth = 2 / 0 = Infinity
// Troika text rendering fails with Infinity outline
```

## Impact
- Text rendering corrupted with Infinity outline
- Low severity: Requires explicitly setting fontSize=0

## Suggested Fix
```typescript
// Line 232 - Use || instead of ?? to catch 0
fontSize: data?.fontSize || 72,

// Or explicit validation
fontSize: (data?.fontSize && data.fontSize > 0) ? data.fontSize : 72,
```
