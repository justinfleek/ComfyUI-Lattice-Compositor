# BUG-146: setCompositionAspect() No Input Validation

## Summary
setCompositionAspect() stores the aspect ratio without validation. NaN, zero, negative, and Infinity values corrupt frustum visualization.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/CameraLayer.ts
- Lines: 132-134
- Function: setCompositionAspect()

## Crash Path Trace
```
Function: setCompositionAspect(aspect: number)

Line 133: this.compositionAspect = aspect;
  - No validation on aspect parameter
  - Value stored directly

Later in createFrustum() at lines 252-254:
  const nearWidth = nearHeight * aspect;  // aspect=NaN → nearWidth=NaN
  const farWidth = farHeight * aspect;    // aspect=NaN → farWidth=NaN

  - aspect=NaN → nearWidth=NaN, farWidth=NaN → frustum vertices all NaN
  - aspect=0 → nearWidth=0, farWidth=0 → degenerate frustum (single line)
  - aspect=-1 → negative dimensions → inverted frustum
  - aspect=Infinity → Infinite frustum dimensions

Result: SILENT CORRUPTION - frustum visualization broken
```

## Code Evidence
```typescript
// Line 132-134
setCompositionAspect(aspect: number): void {
  this.compositionAspect = aspect;  // No validation
}

// Line 249-254 (in createFrustum)
const aspect = this.compositionAspect;

const nearHeight = 2 * Math.tan(fov / 2) * near;
const nearWidth = nearHeight * aspect;  // Corrupted if aspect invalid
const farHeight = 2 * Math.tan(fov / 2) * far;
const farWidth = farHeight * aspect;    // Corrupted if aspect invalid
```

## Impact
- Frustum visualization becomes invisible or glitchy
- Editor display broken
- No error message to indicate cause

## Suggested Fix
```typescript
setCompositionAspect(aspect: number): void {
  if (!Number.isFinite(aspect) || aspect <= 0) {
    console.warn(`setCompositionAspect: invalid aspect ${aspect}, using 16/9`);
    this.compositionAspect = 16 / 9;
    return;
  }
  this.compositionAspect = aspect;
}
```
