# BUG-008: Division by Zero in temporalJitter()

## Summary
The `temporalJitter()` function calculates `period = 1 / frequency` without checking if frequency is zero.

## Severity: LOW

## Location
- File: /ui/src/services/expressions/jitterExpressions.ts
- Lines: 79
- Function: temporalJitter (inner smoothNoise)

## Code Evidence
```typescript
// Line 79
const period = 1 / frequency;  // Division by zero if frequency === 0
```

## Impact
- Returns NaN/Infinity if frequency=0
- Animation becomes corrupted
- Low severity because user must explicitly pass 0

## Suggested Fix
```typescript
const period = frequency > 0 ? 1 / frequency : 1;
```

---

## Fix Applied
**Date:** 2025-12-27
**Fixed by:** Claude (Forensic Audit)

### Code Change
```typescript
// ADDED after line 75 (after const t = time ?? ctx.time):
// Guard against invalid frequency (BUG-008: division by zero)
if (!Number.isFinite(frequency) || frequency <= 0) {
  frequency = 5; // Use default frequency
}
```

### Verification
| Input | Before | After |
|-------|--------|-------|
| frequency=0 | Infinity/wrong values | Uses default 5 ✓ |
| frequency=-1 | Negative period/wrong | Uses default 5 ✓ |
| frequency=NaN | NaN propagates | Uses default 5 ✓ |
| frequency=5 (valid) | Normal jitter | Normal jitter ✓ |

### Pattern Used
SYSTEMIC-003: Division by zero - guard at function start, use default value
