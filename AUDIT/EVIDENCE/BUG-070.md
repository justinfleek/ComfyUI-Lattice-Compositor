# BUG-070: Division by Zero When compositionFPS is 0

## Summary
VideoLayer.setFPS() accepts any number without validation. If fps=0 is set, calculateVideoTime() divides by zero at line 492, producing Infinity video time.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/VideoLayer.ts
- Lines: 556-558 (setFPS), 492 (division)
- Function: setFPS, calculateVideoTime

## The Bug
```typescript
// Lines 556-558 - No validation
setFPS(fps: number): void {
  this.compositionFPS = fps;  // Accepts 0, NaN, negative, Infinity
}

// Line 489-492 - Division by compositionFPS
const layerStartFrame = this.layerData.startFrame ?? 0;
const layerFrame = compositionFrame - layerStartFrame;
const compFps = this.compositionFPS;

let videoTime = (layerFrame / compFps) * effectiveSpeed;  // Division by 0!
```

## Proof of Failure
```typescript
const videoLayer = new VideoLayer(layerData, resources);
videoLayer.setFPS(0);  // No validation

// Later, calculateVideoTime is called
videoLayer.seekToFrame(100);
// Line 492: videoTime = (100 / 0) * 1 = Infinity
// Video seeks to Infinity time - undefined behavior
```

## Impact
- Video playback breaks with Infinity time
- Low severity: Requires explicitly setting fps=0

## Suggested Fix
```typescript
setFPS(fps: number): void {
  if (!Number.isFinite(fps) || fps <= 0) {
    console.warn('VideoLayer: Invalid fps, using 16');
    fps = 16;
  }
  this.compositionFPS = fps;
}
```
