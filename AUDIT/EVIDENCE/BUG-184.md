# BUG-184: NaN Bypasses Normalization in expressionEaseOut()

## Summary
Same vulnerability as BUG-182 and BUG-183 but in expressionEaseOut(). Division by (tMax - tMin) without validation.

## Severity: MEDIUM

## Location
- File: ui/src/services/expressions/expressionEvaluator.ts
- Lines: 219-236
- Function: expressionEaseOut()

## The Bug
```typescript
export function expressionEaseOut(
  t: number,
  tMin: number,
  tMax: number,
  vMin: number | number[],
  vMax: number | number[]
): number | number[] {
  let normalized = (t - tMin) / (tMax - tMin);  // div/0 not protected!
  normalized = Math.max(0, Math.min(1, normalized));  // NaN bypasses!
  // ...
}
```

## Code Evidence
```typescript
// Lines 219-236
export function expressionEaseOut(
  t: number,
  tMin: number,
  tMax: number,
  vMin: number | number[],
  vMax: number | number[]
): number | number[] {
  let normalized = (t - tMin) / (tMax - tMin);
  normalized = Math.max(0, Math.min(1, normalized));

  const eased = 1 - Math.pow(1 - normalized, 3);

  if (Array.isArray(vMin) && Array.isArray(vMax)) {
    return vMin.map((v, i) => v + (vMax[i] - v) * eased);
  }

  return (vMin as number) + ((vMax as number) - (vMin as number)) * eased;
}
```

## Impact
Same as BUG-182/183 - NaN corrupts eased animation values.

## Suggested Fix
Same pattern as BUG-182 fix.
