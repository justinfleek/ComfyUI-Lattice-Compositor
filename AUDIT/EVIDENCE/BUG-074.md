# BUG-074: Division by Zero in ModelLayer Animation Update

## Summary
In ModelLayer.onEvaluateFrame(), the animation delta time is calculated as `1 / this.compositionFps`. If compositionFps is 0, this produces Infinity, causing animation timing corruption.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/ModelLayer.ts
- Line: 783
- Function: onEvaluateFrame

## The Bug
```typescript
// Lines 782-785
if (this.mixer && this.modelData.animation?.autoPlay) {
  const deltaTime = 1 / this.compositionFps;  // Division by zero if fps=0!
  this.updateAnimation(deltaTime);
}
```

The compositionFps comes from BaseLayer which doesn't validate for 0.

## Proof of Failure
```typescript
const modelLayer = new ModelLayer(layerData);
modelLayer.setCompositionFps(0);  // Or via project config

// During frame evaluation
modelLayer.evaluateFrame(10);
// Line 783: deltaTime = 1 / 0 = Infinity
// updateAnimation(Infinity) - animation jumps to end or corrupts
```

## Impact
- Animation timing corrupted with Infinity delta
- Model animations break
- Low severity: Requires fps=0 (uncommon configuration)

## Related: SYSTEMIC-003
Division by fps without validation - same pattern as BUG-067, BUG-070, BUG-071, BUG-073.

## Suggested Fix
```typescript
if (this.mixer && this.modelData.animation?.autoPlay) {
  const fps = this.compositionFps || 30;  // Use || to catch 0
  const deltaTime = 1 / fps;
  this.updateAnimation(deltaTime);
}
```
