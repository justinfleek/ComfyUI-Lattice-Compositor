# BUG-029: extractChannelAmplitudes fps=0/NaN Causes Division by Zero

## Summary
The `extractChannelAmplitudes` function divides by `fps` without validation. When fps=0, produces Infinity. When fps=NaN, produces NaN. Both cause silent failures where no amplitude data is extracted.

## Severity: MEDIUM

## Location
- File: /ui/src/stores/actions/audioActions.ts
- Lines: 529
- Function: `extractChannelAmplitudes`

## The Bug
Line 529 divides `sampleRate` by `fps` without checking if fps is 0 or NaN.

## Code Evidence
```typescript
// Line 529
const samplesPerFrame = Math.floor(sampleRate / fps);  // DIV/0 or NaN

// Line 541-543
for (let frame = 0; frame < frameCount; frame++) {
  const startSample = frame * samplesPerFrame;  // Infinity or NaN
  const endSample = Math.min(startSample + samplesPerFrame, leftData.length);
```

## Crash Path Trace
```
1. Project file corrupted: fps=0
2. loadAudio() -> loadAndAnalyzeAudio(file, fps=0)
3. convertAudioToKeyframes() -> extractChannelAmplitudes(buffer, frameCount, fps=0)
4. Line 529: samplesPerFrame = Math.floor(48000 / 0) = Infinity
5. Line 541: startSample = 0 * Infinity = NaN
6. Line 550: Inner loop never runs (NaN < NaN = false)
7. Line 556: count=0, leftRms=0
8. Returns all-zero arrays -> SILENT DATA LOSS
```

## Impact
- Audio amplitude data silently becomes all zeros
- Keyframes contain no useful data
- No error or warning displayed
- Expressions referencing audio always return 0

## Suggested Fix
```typescript
// At start of extractChannelAmplitudes:
if (!Number.isFinite(fps) || fps <= 0) {
  console.warn('[Audio] Invalid fps:', fps, '- using default 30');
  fps = 30;
}
```
