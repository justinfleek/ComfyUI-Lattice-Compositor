# BUG-067: Division by Zero in simulateToFrame When fps=0

## Summary
The simulateToFrame() method divides by the fps parameter without validation. If fps=0 is passed, deltaTime becomes Infinity, corrupting the entire simulation.

## Severity: LOW

## Location
- File: /ui/src/engine/particles/GPUParticleSystem.ts
- Line: 1502
- Function: simulateToFrame

## The Bug
```typescript
// Lines 1500-1502
simulateToFrame(targetFrame: number, fps: number = 16): number {
  if (!this.frameCacheSystem) return 0;
  const deltaTime = 1 / fps;  // Division by zero if fps=0!
```

While fps has a default value of 16, callers can explicitly pass 0:
```typescript
particleSystem.simulateToFrame(100, 0);  // deltaTime = Infinity
```

## Proof of Failure
```typescript
const system = new GPUParticleSystem({ maxParticles: 1000 });
system.initialize(renderer);

// Explicit fps=0 bypasses the default
system.simulateToFrame(100, 0);

// Line 1502: deltaTime = 1 / 0 = Infinity
// Line 1514: this.step(Infinity) - simulation explodes
// All particle positions become Infinity or NaN
```

## Impact
- Simulation state corrupted with Infinity values
- Particles fly off to infinity
- Low severity: Requires explicitly passing fps=0

## Related: SYSTEMIC-003
Same pattern as other fps division bugs in the codebase.

## Suggested Fix
```typescript
simulateToFrame(targetFrame: number, fps: number = 16): number {
  if (!this.frameCacheSystem) return 0;
  // Validate fps - use || to catch 0, NaN, and negative
  const safeFps = fps > 0 && Number.isFinite(fps) ? fps : 16;
  const deltaTime = 1 / safeFps;
```
