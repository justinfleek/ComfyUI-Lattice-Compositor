# BUG-131: PathLayer.setResolution() No Validation

## Summary
The `setResolution(width, height)` method passes dimensions directly to LineMaterial resolution without validation. Invalid values corrupt line rendering.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/PathLayer.ts
- Lines: 425-430
- Function: setResolution

## The Bug
```typescript
// Lines 425-430
setResolution(width: number, height: number): void {
  this.resolution.set(width, height);  // No validation!
  if (this.guideLine) {
    (this.guideLine.material as LineMaterial).resolution.set(width, height);
  }
}
```

## Proof of Failure
```typescript
const pathLayer = new PathLayer(layerData);

// Attack: width = 0
pathLayer.setResolution(0, 1080);
// LineMaterial resolution = (0, 1080)
// Line2 calculations produce infinity or NaN

// Attack: width = NaN
pathLayer.setResolution(NaN, 1080);
// resolution = (NaN, 1080)
// Line width calculations fail

// Attack: negative
pathLayer.setResolution(-1920, 1080);
// Inverted resolution - undefined behavior
```

## Impact
- Corrupted line rendering
- Guide line may be invisible or incorrectly sized
- Low severity: Resolution typically comes from canvas resize

## Related
- BUG-110: SplineLayer.setResolution no validation (same pattern)

## Suggested Fix
```typescript
setResolution(width: number, height: number): void {
  // Validate dimensions
  if (!Number.isFinite(width) || !Number.isFinite(height) ||
      width <= 0 || height <= 0) {
    console.warn('PathLayer.setResolution: Invalid dimensions');
    return;
  }

  this.resolution.set(width, height);
  if (this.guideLine) {
    (this.guideLine.material as LineMaterial).resolution.set(width, height);
  }
}
```
