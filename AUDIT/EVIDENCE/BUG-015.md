# BUG-015: Type Mismatch in Continue Mode Returns String

## Summary
The `repeatAfter()` and `repeatBefore()` functions in 'continue' mode check `typeof velocity === 'number'` but don't verify that `value` also matches. When `value` is an array and `velocity` is a number, JavaScript string coercion produces a string instead of a number.

## Severity: LOW

## Location
- File: /ui/src/services/expressions/loopExpressions.ts
- Lines: 84-87 (repeatAfter), 139-142 (repeatBefore)
- Functions: repeatAfter, repeatBefore

## The Bug
```typescript
// Lines 84-87
if (typeof velocity === 'number') {
  return (ctx.value as number) + velocity * elapsed;
  // If ctx.value is [100, 200] and velocity is 5:
  // [100, 200] + 50 = "100,20050" (string coercion!)
}
```

## Proof of Failure
```typescript
const ctx = {
  time: 10,
  fps: 30,
  keyframes: [...],
  value: [100, 200],  // Array!
  velocity: 5         // Number!
};

const result = repeatAfter(ctx, 'continue');
console.log(result);       // "100,20050"
console.log(typeof result);  // "string"
```

## Impact
- Returns string instead of number[]
- Type safety violated at runtime
- Downstream calculations may produce unexpected results
- Low severity because requires mismatched types (likely a caller error)

## Suggested Fix
```typescript
case 'continue': {
  const velocity = ctx.velocity;
  const value = ctx.value;

  // Ensure types match
  if (typeof velocity === 'number' && typeof value === 'number') {
    return value + velocity * elapsed;
  }
  if (Array.isArray(velocity) && Array.isArray(value)) {
    return value.map((v, i) => v + (velocity[i] ?? 0) * elapsed);
  }
  // Type mismatch - return unchanged
  console.warn('[Expressions] Type mismatch between value and velocity');
  return value;
}
```
