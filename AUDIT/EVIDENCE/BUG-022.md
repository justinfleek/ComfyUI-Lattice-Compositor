# BUG-022: SHG_TABLE Array Out of Bounds at radius=255

## Summary
The StackBlur implementation clamps radius to 0-255, but SHG_TABLE only has 255 entries (indices 0-254). When radius=255, `SHG_TABLE[255]` returns undefined, causing incorrect blur calculations.

## Severity: LOW

## Location
- File: /ui/src/services/effects/blurRenderer.ts
- Lines: 392-409 (SHG_TABLE definition), 451-452 (radius clamping), 479, 626 (table access)
- Functions: stackBlurHorizontal, stackBlurVertical

## The Bug
```typescript
// Line 451-452 - Clamps to 0-255
radiusX = Math.max(0, Math.min(255, Math.round(radiusX)));

// Line 479 - Accesses table
const shgSum = SHG_TABLE[radius];  // SHG_TABLE[255] = undefined!

// Lines 564, 710 - Uses undefined shift
pixels[pOffset] = (rSum * mulSum) >>> shgSum;  // >>> undefined = >>> 0
```

The SHG_TABLE array definition (lines 392-409) only contains 255 entries, not 256:
- 15 lines Ã— 16 entries = 240
- Last line has 15 entries
- Total: 255 entries (indices 0-254)

## Proof of Failure
```typescript
stackBlur(imageData, 255, 255);
// radius = 255
// shgSum = SHG_TABLE[255] = undefined
// (rSum * mulSum) >>> undefined
// JavaScript converts undefined to 0 for >>> operator
// Result: >>> 0 instead of >>> 24 (expected for large radius)
// Pixel values will be much larger than intended, causing white/blown-out output
```

## Impact
- Incorrect blur output when radius=255
- Pixel values not properly scaled down
- Visual corruption (likely blown-out/white pixels)
- Low severity because 255 is maximum edge case

## Suggested Fix
Change radius clamp from 255 to 254:
```typescript
radiusX = Math.max(0, Math.min(254, Math.round(radiusX)));
radiusY = Math.max(0, Math.min(254, Math.round(radiusY)));
```

Or add 256th entry to SHG_TABLE (value 24).
