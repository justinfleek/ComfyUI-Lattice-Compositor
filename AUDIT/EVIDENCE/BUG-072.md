# BUG-072: NaN Propagation When autoAdvanceSpeed is Infinity

## Summary
In CameraLayer path following, if autoAdvanceSpeed is Infinity, the modulo operation produces NaN which propagates through Math.max/min clamp (SYSTEMIC-005 pattern).

## Severity: LOW

## Location
- File: /ui/src/engine/layers/CameraLayer.ts
- Lines: 508, 522
- Function: applyPathFollowing

## The Bug
```typescript
// Line 505-508
if (pathFollowing.autoAdvance) {
  // Auto-advance mode: calculate t from frame
  t = (frame * pathFollowing.autoAdvanceSpeed) % 1;  // Infinity % 1 = NaN!
}

// Line 521-522 - clamp doesn't catch NaN
t = Math.max(0, Math.min(1, t));  // Math.max(0, Math.min(1, NaN)) = NaN
```

JavaScript behavior:
- `Infinity % 1` = NaN
- `Math.min(1, NaN)` = NaN
- `Math.max(0, NaN)` = NaN

## Proof of Failure
```typescript
const cameraLayer = new CameraLayer(layerData);
cameraLayer.setCameraCallbacks(getter, updater);
cameraLayer.setSplineProvider(provider);

// Set pathFollowing with Infinity speed
layerData.data.pathFollowing = {
  enabled: true,
  autoAdvance: true,
  autoAdvanceSpeed: Infinity,  // User error or corrupted data
  pathLayerId: 'path1',
  // ...
};

// Evaluate frame
cameraLayer.onEvaluateFrame(10);
// Line 508: t = (10 * Infinity) % 1 = NaN
// Line 522: t = Math.max(0, Math.min(1, NaN)) = NaN
// Line 525: splineProvider('path1', NaN, 10) - undefined behavior
```

## Impact
- Camera path following produces undefined position
- Low severity: Requires Infinity autoAdvanceSpeed (rare)

## Related: SYSTEMIC-005
Math.max/min don't catch NaN - same pattern as other bugs.

## Suggested Fix
```typescript
if (pathFollowing.autoAdvance) {
  const speed = Number.isFinite(pathFollowing.autoAdvanceSpeed)
    ? pathFollowing.autoAdvanceSpeed
    : 1;
  t = (frame * speed) % 1;
}
```
