# BUG-076: Infinite Loops with Infinity Tile/Slat Parameters

## Summary
In ProceduralMatteLayer, the checkerboard and venetian blinds patterns use for loops bounded by user-provided tilesX/tilesY/slats values. If these are Infinity, the loops run forever.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/ProceduralMatteLayer.ts
- Lines: 399-405 (checkerboard), 532 (venetian blinds)
- Functions: renderCheckerboard, renderVenetianBlinds

## The Bug
```typescript
// Lines 399-405 - Checkerboard
for (let y = -1; y <= tilesY + 1; y++) {
  for (let x = -1; x <= tilesX + 1; x++) {  // tilesX=Infinity → infinite loop!
    // ...
  }
}

// Line 532 - Venetian blinds
for (let i = 0; i < slats; i++) {  // slats=Infinity → infinite loop!
  // ...
}
```

While defaults are provided (lines 386-387: `tilesX ?? 8`, `tilesY ?? 8`, line 520: `slats ?? 10`), the ternary operator `? ... : default` doesn't catch Infinity:
```typescript
const tilesX: number = params.tilesX ? this.matteEvaluator.evaluate(params.tilesX, frame) : 8;
// If params.tilesX exists and evaluates to Infinity, tilesX = Infinity
```

## Proof of Failure
```typescript
const matteLayer = new ProceduralMatteLayer({
  data: {
    patternType: 'checkerboard',
    parameters: {
      tilesX: { value: Infinity },  // Corrupt or malicious data
    },
    // ...
  }
});

matteLayer.evaluateFrame(0);
// Line 400: for (x = -1; x <= Infinity + 1; x++) - INFINITE LOOP!
// Browser hangs, tab crashes
```

## Impact
- Browser tab hangs/crashes
- Denial of service via corrupt project file
- Low severity: Requires explicitly setting Infinity (rare)

## Related: SYSTEMIC-006
Unbounded loop parameters - same pattern as BUG-059, BUG-061, BUG-068.

## Suggested Fix
```typescript
const tilesX = Math.min(params.tilesX ? this.matteEvaluator.evaluate(params.tilesX, frame) : 8, 1000);
const tilesY = Math.min(params.tilesY ? this.matteEvaluator.evaluate(params.tilesY, frame) : 8, 1000);
// ... same for slats
```
