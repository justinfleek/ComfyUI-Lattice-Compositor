# BUG-136: setMaterialBlendMode() Crashes on Null Material

## Summary
setMaterialBlendMode() has no null check on the `material` parameter. Passing null/undefined causes immediate TypeError crash at line 20.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/blendModeUtils.ts
- Lines: 18-20
- Function: setMaterialBlendMode()

## Crash Path Trace
```
Function: setMaterialBlendMode(material: THREE.Material, mode: string)

Line 18: export function setMaterialBlendMode(material: THREE.Material, mode: string): void {
  - No validation on material parameter

Line 20: material.blending = THREE.NormalBlending;
  - material=null → TypeError: Cannot set properties of null (setting 'blending')
  - material=undefined → TypeError: Cannot set properties of undefined (setting 'blending')

Result: CRASH - TypeError
```

## Code Evidence
```typescript
// Line 18-26
export function setMaterialBlendMode(material: THREE.Material, mode: string): void {
  // Reset to defaults first
  material.blending = THREE.NormalBlending;  // CRASH HERE if material is null
  material.blendEquation = THREE.AddEquation;
  material.blendSrc = THREE.SrcAlphaFactor;
  material.blendDst = THREE.OneMinusSrcAlphaFactor;
  material.blendEquationAlpha = THREE.AddEquation;
  material.blendSrcAlpha = THREE.OneFactor;
  material.blendDstAlpha = THREE.OneMinusSrcAlphaFactor;
```

## Impact
- Unhandled crash if caller passes null material
- Stack trace points to blendModeUtils.ts, not caller's error
- No defensive coding

## Suggested Fix
```typescript
export function setMaterialBlendMode(material: THREE.Material, mode: string): void {
  if (!material) {
    console.warn('setMaterialBlendMode: material is null/undefined');
    return;
  }
  // ... rest of function
}
```
