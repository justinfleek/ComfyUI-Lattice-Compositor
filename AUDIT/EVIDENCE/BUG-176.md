# BUG-176: Division by Zero in periodic()

## Summary
The `periodic()` function divides by `period` without validation, causing NaN when period=0.

## Severity: MEDIUM

## Location
- File: ui/src/services/expressions/expressionEvaluator.ts
- Lines: 61-63
- Function: timeExpressions.periodic()

## The Bug
```typescript
periodic(time: number, period: number): number {
  return (time % period) / period;  // No validation!
}
```

When `period=0`:
- `time % 0` = NaN (JavaScript behavior)
- `NaN / 0` = NaN
- NaN propagates through all calculations using this value

## Code Evidence
```typescript
// Line 61-63
periodic(time: number, period: number): number {
  return (time % period) / period;
}
```

## Impact
- User expressions using `periodic(time, 0)` get NaN
- NaN corrupts animation values silently
- No error shown to user

## Suggested Fix
```typescript
periodic(time: number, period: number): number {
  if (!Number.isFinite(period) || period === 0) return 0;
  return (time % period) / period;
}
```

## Test Case
```javascript
timeExpressions.periodic(5, 0)   // Returns NaN, should return 0
timeExpressions.periodic(5, NaN) // Returns NaN, should return 0
```
