# BUG-119: VideoLayer.setStartTime() No Validation

## Summary
The `setStartTime(time)` method accepts the time parameter without validation. Negative, NaN, or Infinity values are stored and used in `calculateVideoTime()`, causing incorrect video playback positions.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/VideoLayer.ts
- Lines: 606-608
- Function: setStartTime

## The Bug
```typescript
// Lines 606-608
setStartTime(time: number): void {
  this.videoData.startTime = time;  // No validation!
}

// Used in calculateVideoTime() at line 495:
videoTime += this.videoData.startTime;
// If startTime is NaN: videoTime becomes NaN
// If startTime is negative: videoTime can become negative
// If startTime is Infinity: videoTime becomes Infinity
```

## Proof of Failure
```typescript
const videoLayer = new VideoLayer(layerData, resources);
await videoLayer.loadVideo(assetId);

// Attack 1: Negative start time
videoLayer.setStartTime(-5);
// calculateVideoTime: videoTime = ... + (-5)
// May produce negative video time
// Video element: currentTime = negative (invalid)

// Attack 2: NaN start time
videoLayer.setStartTime(NaN);
// calculateVideoTime: videoTime = ... + NaN = NaN
// Video element: currentTime = NaN (undefined behavior)

// Attack 3: Start time > duration
videoLayer.setStartTime(1000);  // Video is only 10 seconds
// calculateVideoTime: videoTime = ... + 1000
// Will be clamped at line 524, but effectiveDuration becomes negative
// effectiveDuration = endTime - startTime = 10 - 1000 = -990
// Loop calculations fail with negative duration
```

## Impact
- Negative start time causes invalid video positions
- NaN propagates through all time calculations
- Start time > duration causes negative effectiveDuration
- Low severity: Usually set from UI with validation, but direct API calls possible

## Related
- BUG-116: setFPS similar no-validation pattern
- 14A: Numeric inputs - negative, NaN, Infinity testing
- 14G: Boundary conditions - values outside expected range

## Suggested Fix
```typescript
setStartTime(time: number): void {
  // Validate time
  if (!Number.isFinite(time) || time < 0) {
    console.warn('VideoLayer.setStartTime: Invalid time, using 0');
    time = 0;
  }
  // Clamp to video duration if metadata available
  if (this.metadata && time > this.metadata.duration) {
    time = this.metadata.duration;
  }
  this.videoData.startTime = time;
}
```
