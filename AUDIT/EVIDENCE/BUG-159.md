# BUG-159: LightLayer.setAreaSize() No Validation

## Summary
setAreaSize() sets RectAreaLight width/height without validation. Zero, NaN, negative, or Infinity dimensions create invalid area lights.

## Severity: LOW

## Location
- File: /ui/src/engine/layers/LightLayer.ts
- Lines: 680-687
- Function: setAreaSize()

## Crash Path Trace
```
Function: setAreaSize(width: number, height: number)

Line 681: if (this.light instanceof THREE.RectAreaLight) {
Line 682: this.lightData.areaWidth = width;   // NaN stored
Line 683: this.lightData.areaHeight = height; // NaN stored
Line 684: this.light.width = width;   // RectAreaLight.width = NaN
Line 685: this.light.height = height; // RectAreaLight.height = NaN

Test cases:
  - width=NaN → light.width=NaN → area light calculations fail
  - width=0 → zero-area light (no illumination)
  - width=-100 → negative area (undefined behavior)
  - width=Infinity → infinite area (memory/calculation issues)

Result: SILENT CORRUPTION - area light dimensions invalid
```

## Code Evidence
```typescript
// Lines 680-687
setAreaSize(width: number, height: number): void {
  if (this.light instanceof THREE.RectAreaLight) {
    this.lightData.areaWidth = width;   // No validation
    this.lightData.areaHeight = height; // No validation
    this.light.width = width;
    this.light.height = height;
  }
}
```

## Impact
- NaN dimensions break area light rendering
- Zero dimensions create non-functional light
- Negative dimensions have undefined Three.js behavior
- Area light stops illuminating scene correctly

## Suggested Fix
```typescript
setAreaSize(width: number, height: number): void {
  if (!Number.isFinite(width) || !Number.isFinite(height) || width <= 0 || height <= 0) {
    console.warn(`setAreaSize: invalid dimensions ${width}x${height}`);
    return;
  }
  // ...
}
```
