# BUG-099: GPUParticleSystem.initialize() Double Call Not Prevented

## Summary
The `initialize()` method creates GPU resources (geometry, material, textures, GPU physics) without checking if already initialized. Calling initialize() twice creates duplicate resources without disposing the previous ones, causing memory leaks.

## Severity: LOW

## Location
- File: /ui/src/engine/particles/GPUParticleSystem.ts
- Lines: 417-444
- Function: initialize

## The Bug
```typescript
// Lines 417-444
initialize(renderer: THREE.WebGLRenderer): void {
  this.renderer = renderer;
  this.gl = renderer.getContext() as WebGL2RenderingContext;
  // NO CHECK for already initialized!

  // Create Three.js mesh for rendering (creates material)
  this.createParticleMesh();  // Creates new geometry and material

  // Create modulation textures
  this.createModulationTextures();  // Creates new textures

  // Initialize trail system if enabled
  if (this.config.render.trailLength > 0) {
    this.initializeTrails();  // Creates new trail system
  }

  // Initialize GPU physics system
  this.gpuPhysics = new ParticleGPUPhysics({...});  // Creates new GPU resources
  this.gpuPhysics.initialize(renderer, ...);
}
```

## Proof of Failure
```typescript
const system = new GPUParticleSystem(config);

// First initialization - creates resources
system.initialize(renderer);
// Creates: instancedGeometry, material, textures, gpuPhysics

// Accidental second initialization (e.g., component re-mount)
system.initialize(renderer);
// Creates NEW: instancedGeometry, material, textures, gpuPhysics
// Old resources are orphaned - memory leak!

// After 10 re-initializations:
// - 10 orphaned geometries
// - 10 orphaned materials
// - 10 orphaned texture sets
// - 10 orphaned GPU physics instances
// GPU memory continuously grows
```

## Impact
- Memory leak with each re-initialization
- GPU memory exhaustion over time
- Performance degradation
- Low severity: Requires explicit double initialization call

## Related
- BUG-094: BaseLayer.dispose() memory leaks (similar pattern)
- 14I: Ordering/Timing - init() called twice

## Suggested Fix
```typescript
private initialized = false;

initialize(renderer: THREE.WebGLRenderer): void {
  if (this.initialized) {
    console.warn('GPUParticleSystem.initialize: Already initialized');
    return;
  }

  this.renderer = renderer;
  this.gl = renderer.getContext() as WebGL2RenderingContext;

  if (!this.gl) {
    throw new Error('WebGL2 context required for GPU particle system');
  }

  // ... rest of initialization

  this.initialized = true;
}

// Also add isInitialized() accessor
isInitialized(): boolean {
  return this.initialized;
}
```
